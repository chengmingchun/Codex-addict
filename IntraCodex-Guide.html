Intra Codex

公司内网 Agent 工作台使用说明

Intra Codex 是一个类似 Codex 的桌面工具。它不直接绑定某一个模型，而是通过可配置的 CLI 后端调用 opencode、cac、codeagent 等内部工具，帮助团队并发执行代码分析、修改、审查和自动化任务。

入口 直接运行编译好的 intra-codex.exe
默认主题 浅色主题，可在左侧工具栏切换深色
配置位置 config/providers.yaml 和 skills/*.md

核心特性

多后端 CLI 同一个界面里切换 cac、opencode、codeagent，也可以增加新的内部 wrapper。
并发任务队列 支持全局并发限制和每个 provider 的并发限制，适合内部额度或资源受控场景。
Skills 注入 把团队常用工作流写成 Markdown，发任务时勾选后自动拼进 prompt。
实时输出流 CLI 的 stdout、stderr、退出码会显示在任务输出流里，方便排查失败原因。
额度用量看板 展示运行中、排队中、今日完成和粗略 token 用量，后续可接真实计费接口。
单 exe 运行 当前版本使用 Tauri 构建，直接运行 exe，不需要安装包。

怎么使用

选择执行后端 在“执行后端”下拉框里选择 cac、opencode、codeagent 或 local-demo。
填写工作目录 默认是 .，相对于 providers.yaml 里的 workspaceRoot。Agent 会在这个目录下执行 CLI。
勾选 Skills 选择适合任务的技能，例如 Frontend Builder 或 Code Review。勾选的内容会被自动放到 prompt 前面。
写任务目标 在输入框里写清楚希望 Agent 完成什么。建议包含目标、限制、验证方式和期望输出。
启动任务 点击“启动任务”后，任务进入队列。左侧会出现任务会话，中间输出区会实时显示命令执行结果。
查看或停止 任务运行时可以点击“停止”。完成后可以在左侧切换历史任务查看输出。

Provider 配置

每个 provider 对应一个可执行 CLI。公司内部可以把 cac 或 codeagent 配在这里。

providers:
  - id: cac
    label: CAC OpenCode Wrapper
    command: cac
    args:
      - opencode
      - run
    concurrency: 3
    cwd: "."
    shell: true
    env:
      CAC_PROFILE: default
defaults:
  providerId: cac
  maxGlobalConcurrency: 3
  workspaceRoot: ".."

Skills 配置

把 Markdown 文件放进 skills 目录即可。文件名会作为 skill id，一级标题会显示在界面上。

# Code Review

用于代码审查任务。
优先关注 bug、回归风险、遗漏测试和可维护性问题。
输出按严重程度排序。

界面区域说明

区域 作用
左侧任务会话 显示所有任务的状态。点击某个任务可以查看它的输出流。
中间发起任务 选择 provider、填写目录、勾选 skills、输入任务描述并启动。
输出流 实时显示 system、stdout、stderr。失败时优先看 stderr 和退出码。
右侧额度用量 展示运行、排队、完成和 token 粗估。后续可替换为真实额度数据。
左侧工具栏 刷新配置、查看 provider 配置入口、切换浅色/深色主题。

常见问题

任务一启动就失败

  • 检查 command 是否在系统 PATH 里。
  • 检查 args 是否符合内部 CLI 的真实参数。
  • 如果是 Windows wrapper，通常 shell: true 更稳。

为什么有 local-demo

  • 它只用于本机 smoke test。
  • 没有接入真实 cac 前，可以用它验证队列和输出流。
  • 正式分发时可以从 providers.yaml 删除。

怎么控制并发

  • maxGlobalConcurrency 控制全局同时运行任务数。
  • 每个 provider 的 concurrency 控制它自己的并发。
  • 例如 cac 大约并发 3 个，就配置 concurrency: 3。

怎么换主题

  • 默认打开是浅色。
  • 左侧工具栏的太阳/月亮按钮可切换浅色和深色。
  • 选择会保存在本机浏览器存储里。

建议团队内部先统一 provider 命令规范，再沉淀常用 skills。这样这个工具会更像“公司内部 Codex 工作台”，而不是一个普通命令启动器。
