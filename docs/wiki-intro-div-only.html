Intra Codex / Agent Workbench

把内网 CLI Agent
组织成可沉淀的研发工作台

Intra Codex 不是替代现有 Agent，而是把 opencode、cac、codeagent 等命令行能力统一收束到项目、会话、Skills、上下文文件和运行记录里，让 AI 辅助研发变得可复用、可追踪、可交付。

项目级会话
Skills 注入
上下文文件选择
离线 Markdown 渲染
Windows EXE 分发
$ open project ./service-console
✓ auto create session
✓ load skills: explore, html-report
✓ select context files
→ Ctrl + Enter to run agent

它解决什么问题？

Context

上下文不再散落

围绕项目目录组织文件树、会话历史和上下文选择，减少每次都要重新解释项目背景的成本。

Workflow

Skills 变成工作流资产

把探索项目、输出 HTML 报告等固定流程沉淀为 Skill，让团队成员按同一标准使用 Agent。

Delivery

结果适合内网沉淀

Markdown 本地渲染，HTML 报告不依赖 CDN 和在线引擎，适合复制到内网 Wiki。

推荐使用流程

1

打开项目

选择本地代码目录，应用自动读取文件树。

2

自动建会话

无需手动创建 Session，打开项目后直接进入新会话。

3

选择 Skills

默认使用探索项目和 HTML 探索报告两个 Skill。

4

勾选上下文

选择 README、配置、入口和核心 service 等关键文件。

5

运行 Agent

输入任务后使用 Ctrl/⌘ + Enter 发送。

默认 Skills

Skill 01

探索项目

先理解项目定位、模块地图、核心链路、关键文件和风险点，再给出下一步建议。

Skill 02

HTML 探索报告

把探索结果整理成可离线打开、适合粘贴到 Wiki 的纯 HTML/CSS 报告。

Offline

不依赖外网

不使用在线字体、在线图片、CDN、PlantUML server 或 Mermaid CDN。

Wiki 嵌入说明：本介绍页不包含任何 JavaScript，不包含 script 标签。视觉效果只使用 div 结构和 CSS 动画实现，适合复制到支持 HTML/CSS 的内网 Wiki 页面。
