# HTML 探索报告

用于把探索结果整理成一个可离线打开的 HTML 报告。报告必须适合内网环境，不依赖任何 CDN、在线图片、在线字体、PlantUML server、Mermaid CDN 或其他外部资源。

输出要求：

1. 只输出一个完整 HTML 文档，从 `<!doctype html>` 开始。
2. CSS 和 JavaScript 必须内联在 HTML 内。
3. 页面顶部必须有搜索框，支持本地关键词搜索和结果高亮。
4. 每个模块/文件/风险点使用带 `data-search` 的卡片或 section，方便搜索过滤。
5. 内容结构要包括：项目定位、模块地图、核心链路、关键文件、风险点、下一步建议。
6. 如果需要展示图，不要调用在线渲染引擎；可以用纯 HTML/CSS/SVG 或 ASCII 图。
7. 如果无法确认某个结论，必须标注“推测”或“不确定”。

HTML 交互最低要求：

- 搜索框输入时实时过滤报告块。
- 匹配文本要高亮。
- 有“全部展开/收起”或类似的本地交互。
- 页面必须在断网环境下可用。
